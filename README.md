# Анализ логов веб-сервера

Скрипт анализирует лог в формате:

```text
%h - - %t "%r" %s %b "%{Referer}" "%{User-Agent}" %d
```

Поддерживаются два варианта запуска:
- передать путь к одному лог-файлу;
- передать путь к директории, тогда будут обработаны все файлы внутри этой директории.

За один проход по каждому файлу скрипт собирает:
- общее количество запросов;
- количество запросов по HTTP-методам `GET`, `POST`, `PUT`, `DELETE`, `OPTIONS`, `HEAD`;
- топ-3 IP-адресов по количеству запросов;
- топ-3 самых долгих запросов с методом, URL, IP, датой и длительностью.

## Запуск

Из корня проекта:

```bash
python3 -m src.log_analyzer /path/to/access.log
```

Если нужно указать папку для json-отчётов:

```bash
python3 -m src.log_analyzer /path/to/logs --output-dir reports
```

По умолчанию отчёты сохраняются в директорию `log_reports`.

## Формат результата

Для каждого обработанного файла создаётся отдельный `json`:

```json
{
  "top_ips": {
    "10.0.0.1": 3,
    "10.0.0.2": 2,
    "10.0.0.3": 1
  },
  "top_longest": [
    {
      "ip": "10.0.0.3",
      "date": "[01/Jan/2024:10:00:03 +0000]",
      "method": "DELETE",
      "url": "/item/1",
      "duration": 780
    }
  ],
  "total_stat": {
    "GET": 2,
    "POST": 1,
    "PUT": 1,
    "DELETE": 1,
    "OPTIONS": 1,
    "HEAD": 1
  },
  "total_requests": 7
}
```

Эта же статистика дополнительно печатается в терминал после обработки файла.
